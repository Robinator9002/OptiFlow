Technische Übersicht

Dieser Abschnitt richtet sich an Personen mit IT-Kentnissen und beschreibt die zugrundeliegende Architektur und die verwendeten Technologien des Dateimanagers. Ich beleuchte die Interaktion zwischen dem React-Frontend und dem Uvicorn/FastAPI-Backend, die Datenhaltung, die Netzwerkkommunikation und andere relevante technische Details.

Architektur

Der Dateimanager ist als moderne Webanwendung konzipiert und folgt einer klaren Client-Server-Architektur. Das Frontend ist mit React entwickelt, während das Backend auf Uvicorn und FastAPI basiert. Diese Wahl ermöglicht eine effiziente, skalierbare und wartbare Anwendungsentwicklung.

  • Frontend: Das Frontend ist mit React realisiert und bietet eine dynamische und benutzerfreundliche Oberfläche. Es kommuniziert über HTTP-Anfragen die mit Axios erstellt werden mit dem Backend, um Daten abzurufen und Aktionen auszuführen.
  • Backend: Das Backend ist mit Uvicorn und FastAPI aufgebaut. Uvicorn dient als ASGI-Server (Asynchronous Server Gateway Interface) und ermöglicht die Ausführung von FastAPI, einem modernen, leistungsstarken Python-Framework für den Aufbau von APIs.
  • Datenbank: Die Daten werden in einer lokalen Datenbank gespeichert. Details zum verwendeten Datenbankmanagementsystem und dem Schema werden im Abschnitt "Datenhaltung" erläutert.

Frontend (React)

Das Frontend ist mit React entwickelt, einer beliebten JavaScript-Bibliothek für den Aufbau von Benutzeroberflächen. React ermöglicht die Erstellung von Single-Page-Anwendungen (SPAs), die schnell, reaktionsschnell und interaktiv sind.

  • Komponentenbasierte Architektur: Das Frontend ist in wiederverwendbare Komponenten unterteilt, die die Benutzeroberfläche strukturieren und die Entwicklung erleichtern.
  • Zustandsmanagement: React verwendet ein effizientes Zustandsmanagement, um die Datenflüsse in der Anwendung zu verwalten und die Leistung zu optimieren.
  • Asynchrone Kommunikation: Das Frontend kommuniziert asynchron mit dem Backend, um Daten abzurufen und Aktionen auszuführen, ohne die Benutzeroberfläche zu blockieren.

Backend (Uvicorn, FastAPI)

Das Backend ist mit Uvicorn und FastAPI realisiert. Diese Technologien bieten eine moderne, leistungsstarke und flexible Plattform für die Entwicklung von APIs.

  • Uvicorn: Uvicorn ist ein blitzschneller ASGI-Server, der auf uvloop und httptools basiert. Er ermöglicht die Ausführung von asynchronen Python-Webanwendungen und bietet eine hervorragende Leistung.
  • FastAPI: FastAPI ist ein modernes, schnelles (High-Performance) Web-Framework für den Aufbau von APIs mit Python 3.6+ basierend auf Standard-Python-Typhinting.
    • Typisierung: FastAPI verwendet Python-Typhints, um die Validierung von Daten, die Serialisierung und die Dokumentation der API zu automatisieren.
    • Abhängigkeitsinjektion: FastAPI unterstützt ein leistungsfähiges System zur Abhängigkeitsinjektion, das die Wiederverwendbarkeit von Code und die Testbarkeit verbessert.
    • Automatische Dokumentation: FastAPI generiert automatisch eine interaktive API-Dokumentation (Swagger UI), die die Entwicklung und den Test der API erleichtert.

Kommunikation zwischen Frontend und Backend

Das Frontend und das Backend kommunizieren über HTTP-Anfragen. Das Frontend sendet Anfragen an das Backend, um Daten abzurufen oder Aktionen auszuführen, und das Backend antwortet mit den entsprechenden Daten oder Statusmeldungen.

  • HTTP-Methoden: Die Kommunikation erfolgt über standardmäßige HTTP-Methoden wie GET, POST, PUT, DELETE usw.
  • JSON-Datenformat: Die ausgetauschten Daten sind in der Regel im JSON-Format kodiert, das einfach zu parsen und zu verarbeiten ist.
  • CORS: Cross-Origin Resource Sharing (CORS) ist konfiguriert, um Anfragen von verschiedenen Domänen zu ermöglichen.

Um die Sicherheit zu gewährleisten, wird ein Token-basiertes Authentifizierungssystem verwendet. Nach erfolgreichem Login erhält der Client ein Token, das er bei nachfolgenden Anfragen an den Server sendet. Der Server überprüft das Token, um die Identität des Benutzers zu bestätigen und die Anfrage zu autorisieren.

Datenhaltung

Der Dateimanager speichert Benutzerdaten, Einstellungen und andere relevante Informationen in einer lokalen Datenbank.

  • Lokale Datenbank: Die Verwendung einer lokalen Datenbank reduziert die Abhängigkeit von externen Servern und verbessert die Leistung.
  • Datenbankschema: Das Datenbankschema ist sorgfältig entworfen, um die Integrität und Konsistenz der Daten zu gewährleisten. Es umfasst Tabellen für Benutzer, Einstellungen, Dateien und andere relevante Entitäten.
  • ORM (Object-Relational Mapping): Ein ORM-System wird verwendet, um die Interaktion mit der Datenbank zu vereinfachen und die Entwicklung zu beschleunigen.

Sicherheit

Sicherheit ist ein zentraler Aspekt des Dateimanagers. Es wurden verschiedene Maßnahmen ergriffen, um die Daten und die Anwendung vor Bedrohungen zu schützen.

  • Authentifizierung: Ein Token-basiertes Authentifizierungssystem wird verwendet, um die Identität der Benutzer zu überprüfen und den Zugriff auf geschützte Ressourcen zu sichern.
  • Passwort-Hashing: Passwörter werden mit starken Hashing-Algorithmen verschlüsselt, um sie vor unbefugtem Zugriff zu schützen.
  • Autorisierung: Die Autorisierung erfolgt auf der Grundlage von Benutzerrollen (z.B. Administrator, normaler Benutzer), um den Zugriff auf bestimmte Funktionen und Daten zu steuern.
  • Validierung: Eingaben von Benutzern und anderen Quellen werden sorgfältig validiert, um Sicherheitslücken wie SQL-Injection und Cross-Site Scripting (XSS) zu vermeiden.
  • CORS: CORS ist korrekt konfiguriert, um sicherzustellen, dass nur autorisierte Domänen auf die API zugreifen können.

Technik des Event-Systems

Das Event-System ermöglicht die automatische Ausführung von Aufgaben zu bestimmten Zeiten. Technisch gesehen, funktioniert dies wie folgt:

  • Zeitplanung: Der Benutzer definiert Ereignisse mit einer bestimmten Zeit oder einem Zeitplan. Diese Informationen werden in der Datenbank gespeichert.
  • Scheduler: Ein Hintergrundprozess (Scheduler) überwacht die Datenbank regelmäßig, um festzustellen, ob Ereignisse ausgeführt werden müssen.
  • Ausführung: Wenn ein Ereignis fällig ist, führt der Scheduler die entsprechende Aufgabe aus. Dies kann das Ausführen einer Funktion, das Senden einer Nachricht oder eine andere Aktion sein.
  • Dynamische Anpassung: Der Zeitplan von Ereignissen kann dynamisch geändert werden, ohne dass ein Neustart der Anwendung erforderlich ist.
