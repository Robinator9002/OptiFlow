Installation und Start

Die Inbetriebnahme des OptiFlow Dateimanagers ist unkompliziert und erfordert keine komplexe Server-Einrichtung. Die Anwendung ist als eigenständiges Paket konzipiert, das alle notwendigen Komponenten enthält und auf einem Windows-Server oder einer leistungsstarken Workstation ausgeführt werden kann.

Systemanforderungen

Bevor Sie beginnen, stellen Sie sicher, dass das vorgesehene Serversystem die folgenden Mindestanforderungen erfüllt:

  • Betriebssystem: Windows 10 oder neuer (Windows Server 2016 oder neuer empfohlen).
  • Hardware: Ein moderner Multi-Core-Prozessor und ausreichend Arbeitsspeicher (empfohlen: 8 GB+) für eine performante Indexierung.
  • Webbrowser (für Clients): Eine aktuelle Version von Google Chrome, Mozilla Firefox oder Microsoft Edge auf den Rechnern der Benutzer.
  • Netzwerk: Der Server und die Client-Rechner müssen sich im selben lokalen Netzwerk (LAN) befinden und der Server-Port (standardmäßig Port 8000) muss für die Clients erreichbar sein.

1. Paket herunterladen und entpacken

Sie erhalten die Anwendung als eine einzige ZIP-Datei. Laden Sie diese herunter und entpacken Sie sie an einem geeigneten, permanenten Ort auf dem Server (z.B. C:\OptiFlow\). Vermeiden Sie kurzlebige Pfade wie den Desktop oder Download-Ordner.

Der entpackte Ordner enthält die gesamte Anwendungslogik, einschließlich:

  • Backend-Server (Python/FastAPI)
  • Frontend-Oberfläche (React)
  • Notwendige Hilfsprogramme (z.B. für OCR)
  • Start-Skripte (.bat Dateien)

2. Anwendung starten

Im Hauptverzeichnis der entpackten Anwendung finden Sie mehrere .bat-Dateien. Für den normalen Betrieb ist nur eine davon relevant.

Doppelklicken Sie auf die Datei 'start.bat', um die Anwendung zu initialisieren.

Die wichtigste Datei: start.bat

Ein einfacher Doppelklick auf die start.bat-Datei genügt. Dieses Skript führt die folgenden Aktionen aus:

  1. Es startet den Backend-Server. Ein Konsolenfenster öffnet sich, das die Server-Logs anzeigt. Dieses Fenster muss geöffnet bleiben, solange die Anwendung läuft! Ein Schließen dieses Fensters beendet den Server.
  2. Anschließend startet es die Webanwendung und öffnet diese automatisch im Standard-Webbrowser des Servers.

Zugriff für andere Benutzer

Andere Benutzer im Netzwerk können auf OptiFlow zugreifen, indem sie die IP-Adresse oder den Hostnamen des Servers gefolgt vom Port 8000 in ihrem Browser eingeben. Beispiel: http://192.168.1.10:8000 oder http://fileserver:8000.

Hinweis für Administratoren

Es wird empfohlen, das Start-Skript als geplante Aufgabe (Scheduled Task) bei Systemstart auszuführen zu lassen. Dies stellt sicher, dass OptiFlow nach einem Neustart des Servers automatisch wieder verfügbar ist.

Nächster Schritt

Die Anwendung läuft! Nun ist es an der Zeit, sich zum ersten Mal anzumelden und das System zu konfigurieren. Da noch keine Benutzer existieren, befindet sich die Anwendung im Einrichtungsmodus.
