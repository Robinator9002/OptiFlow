Datenmanagement, Backup & Wiederherstellung

Ein robustes Datenmanagement ist das Rückgrat eines jeden zuverlässigen Systems. Dieser Abschnitt ist der Leitfaden für Administratoren zur Sicherung, Wiederherstellung und im Notfall auch zur Zurücksetzung der OptiFlow-Kerndaten. Ein Verständnis dieser Prozesse ist entscheidend für die Aufrechterhaltung der Systemintegrität.

Der `data` Ordner: Das Herz des Systems

Alle Kerndaten von OptiFlow – Benutzer, Index, Events, etc. – werden als separate .json-Dateien im data-Verzeichnis gespeichert. Der genaue Pfad hängt von Ihrer Installationsmethode ab. Alle hier beschriebenen Aktionen beziehen sich auf die Dateien in diesem Ordner.

1. Backup erstellen

Sichern Sie regelmäßig alle kritischen Systemdateien direkt über die Web-Oberfläche.

  1. Gehen Sie zu Einstellungen → Datenbank.
  2. Wählen Sie nacheinander jede wichtige Datei aus (z.B. users.json, index.json, events.json).
  3. Klicken Sie für jede Datei auf den Button "Download Backup".
  4. Bewahren Sie diese JSON-Dateien an einem sicheren, externen Ort auf.

2. System wiederherstellen

Stellen Sie einen früheren Systemzustand aus Ihren Backup-Dateien wieder her.

  1. Stoppen Sie die OptiFlow-Anwendung vollständig.
  2. Navigieren Sie zum data-Ordner auf Ihrem Server.
  3. Ersetzen Sie die vorhandenen .json-Dateien durch Ihre Backup-Versionen.
  4. Starten Sie OptiFlow neu.
  5. Wichtig: Gehen Sie zu Funktionen (Admin) → Index & Scanner und klicken Sie auf "Index laden", um sicherzustellen, dass die App den wiederhergestellten Index korrekt einliest.

3. Notfall-Reset

Wenn der Admin-Zugang verloren geht, können Sie das Benutzersystem zurücksetzen.

  1. Stoppen Sie die OptiFlow-Anwendung vollständig.
  2. Navigieren Sie zum data-Ordner.
  3. Löschen Sie nur die Datei users.json.
  4. Starten Sie OptiFlow neu.
  5. Die App befindet sich nun im Ersteinrichtungs-Modus. Der nächste Benutzer, der sich registriert, wird automatisch zum neuen Administrator.

Achtung: Der "Passwort vergessen"-Reset ist destruktiv!

Die im Login-Dialog beschriebene und hier unter Punkt 3 detaillierte Methode zum Zurücksetzen eines vergessenen Admin-Passworts ist ein letzter Ausweg.

Durch das Löschen der users.json gehen alle Benutzerkonten, alle Passwörter und alle benutzerspezifischen Einstellungen unwiderruflich verloren. Ihre anderen Daten wie der Suchindex (index.json) bleiben zwar erhalten, aber die gesamte Benutzerverwaltung wird auf null gesetzt.

Dieser Vorgang sollte nur durchgeführt werden, wenn eine Wiederherstellung über ein Backup nicht möglich oder nicht gewünscht ist.
