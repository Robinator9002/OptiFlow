Technische Übersicht

Invertierte Indexierung

Die Grundlage für die blitzschnelle Suche bildet die invertierte Indexierung, eine Technik, die in der Information Retrieval (IR) verwendet wird, um Suchvorgänge zu beschleunigen. Statt die Dokumente sequentiell zu durchsuchen, erstellt diese Methode einen Index, der Wörter und Metadaten auf die Dokumente abbildet, in denen sie vorkommen. Dies ermöglicht es, Suchanfragen mit hoher Effizienz zu beantworten, da nicht jede Datei einzeln durchsucht werden muss.

Der Index wird als Hash-Tabelle oder Baumstruktur implementiert, was den schnellen Zugriff auf die relevanten Dokumente ermöglicht. Bei der Indexierung werden Texte in Token zerlegt, Stoppwörter entfernt und Stemming-Algorithmen angewendet, um die Effektivität der Suche zu erhöhen.

Suchsystem

Das Suchsystem verwendet ein ausgeklügeltes Punktesystem, um die Relevanz der Suchergebnisse zu bewerten. Dabei werden verschiedene Faktoren wie die Häufigkeit des Suchbegriffs (Term Frequency), seine Position im Text (Proximity), Übereinstimmungen mit Metadaten und die inverse Dokumenthäufigkeit (Inverse Document Frequency, IDF) berücksichtigt. Die Ergebnisse werden dynamisch sortiert, um die relevantesten Dateien zuerst anzuzeigen.

Die IDF gewichtet seltene Begriffe höher, da sie oft aussagekräftiger sind. Die Kombination von TF und IDF (TF-IDF) ermöglicht eine präzise Bewertung der Relevanz von Dokumenten für eine bestimmte Suchanfrage.

Dateiverwaltung

Die Dateiverwaltung ermöglicht das direkte Öffnen, Bearbeiten und Löschen von Dateien innerhalb der Anwendung. Eine Vorschaufunktion zeigt Dateiinhalte und Metadaten an, ohne dass die Dateien extern geöffnet werden müssen. Die integrierte Bearbeitungsfunktion unterstützt schnelle Änderungen an Textdateien durch die verwendung von contenteditable und anderen Javascript basierten techniken.

Die Anwendung verwendet APIs des Betriebssystems, um Dateien zu öffnen und zu bearbeiten. Die Vorschaufunktion nutzt Bibliotheken wie react-pdf, um verschiedene Dateiformate anzuzeigen.

Indexierungskonfiguration

Die Indexierung kann flexibel an verschiedene Anforderungen angepasst werden. Benutzer können Dateitypen, Ordner und Größenbeschränkungen definieren, um die Indexierung zu optimieren. Die Konfiguration wird in einer separaten Datei gespeichert und kann jederzeit angepasst werden.

Die Konfigurationsdatei wird im JSON-Format gespeichert, was die einfache Bearbeitung und das Lesen der Einstellungen ermöglicht. Die Anwendung verwendet Bibliotheken wie fs (Node.js) oder os (Python), um auf das Dateisystem zuzugreifen und die Indexierung entsprechend der Konfiguration durchzuführen.

Sicherheit

Sicherheit hat oberste Priorität. Alle Index- und Nutzerdaten werden lokal gespeichert, um externen Zugriff zu verhindern. Die Anwendung läuft auf einem lokalen Server, und ein automatisches Backup-System schützt vor Datenverlust.

Die Anwendung verwendet Verschlüsselungstechniken, um sensible Daten zu schützen. Ein automatisches Backup-System erstellt regelmäßig Sicherungskopien des Index und der Konfigurationsdateien, um Datenverlust zu verhindern.

Dieses Diagramm zeigt, wie alle Dateien lokal auf dem Server abgespeichert werden, sodass selbst bei einem Absturz oder Schließen der Anwendung ein Backup vorhanden ist. Bei Änderungen in der Dateimanager Klasse (die den Index enthält) wird automatisch die file_index.json-Datei, die den Index lokal abspeichert, aktualisiert. Außerdem wird beim Schließen und Neu Öffnen der App (des Servers, nicht der clientbasierten Anwendung) automatisch die file_index.json-Datei ausgelesen, sodass sofort wieder alle Daten vorhanden sind.

Kommunikation zwischen Backend und Frontend

Die Anwendung ist in zwei Hauptkomponenten unterteilt: ein clientbasiertes Frontend und ein Server-Backend. Das Frontend, entwickelt in React, stellt die Benutzeroberfläche dar und läuft unabhängig auf den Computern der Mitarbeiter. Es ermöglicht die Interaktion mit dem Dateimanager, wie das Ausführen von Suchen und das Anzeigen von Ergebnissen. Das Backend, entwickelt in Python mit FastAPI, verwaltet die gesamte Logik, Indexierung und Suche. Es läuft auf einem zentralen Server und ist für die Verarbeitung von Suchanfragen, die Verwaltung des Index und die Interaktion mit dem Dateisystem verantwortlich.

Die Kommunikation zwischen Frontend und Backend erfolgt über eine RESTful API, die von FastAPI bereitgestellt wird. React verwendet Axios, um HTTP-Anfragen an das Backend zu senden und die empfangenen Daten darzustellen.

Die API verwendet JSON als Datenaustauschformat, was die Interoperabilität zwischen Frontend und Backend erleichtert. FastAPI bietet automatische Validierung und Serialisierung von Daten, was die Entwicklung und Wartung der API vereinfacht.

Wichtige Begriffe:

  • Frontend (Client): Die Benutzeroberfläche der Anwendung, die auf den Computern der Mitarbeiter läuft.
  • Backend (Server): Die serverseitige Logik, die auf einem zentralen Server läuft und die Datenverarbeitung übernimmt.
  • RESTful API: Eine Schnittstelle für die Kommunikation zwischen Frontend und Backend.
  • FastAPI: Ein modernes, schnelles Web-Framework für Python zur Erstellung von APIs.
  • React: Eine JavaScript-Bibliothek zur Erstellung von Benutzeroberflächen.
  • Axios: Eine JavaScript-Bibliothek für HTTP-Anfragen.
  • JSON: Ein leichtes Datenaustauschformat.
  • TF-IDF: Eine Methode zur Bewertung der Relevanz von Dokumenten für Suchanfragen.
